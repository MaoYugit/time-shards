# 项目分析与个人开发路线图

基于对 `time-shards` (时光碎片) 项目代码和结构的分析，以下是关于该项目功能的详细拆解以及个人开发的推荐路径。

## 1. 项目分析 (目前具备的功能)

`time-shards` 是一个现代化的、视觉效果丰富的个人博客及知识库系统。

### 核心技术栈
*   **前端框架**: Vue 3 (Composition API) + Vite (构建工具)。
*   **状态管理**: Pinia (管理全局设置，如主题、特效)。
*   **路由**: Vue Router (标准的单页应用路由)。
*   **视觉特效**: Three.js (3D 背景), GSAP (动画), TSParticles (粒子特效)。
*   **编辑器**: Vditor / Markdown-it (Markdown 编辑与渲染)。
*   **国际化**: Vue I18n (多语言支持)。

### 主要功能模块
*   **用户系统**:
    *   登录 (`Login.vue`) 与 注册 (`Register.vue`)。
    *   个人资料管理 (`Profile.vue`) 与 系统设置 (`Settings.vue`)。
    *   基于 Token 的身份验证（从 `todo.md` 中关于 Token 过期的讨论可以看出）。
*   **内容管理**:
    *   **博客/文章系统**: 首页布局 (`Home.vue`)、文章详情 (`ArticleDetail.vue`)、归档 (`Archive.vue`)。
    *   **编辑器**: 富文本 Markdown 编辑器 (`ArticleEditor.vue`)，支持预览和语法高亮 (`highlight.js`, `katex`)。
    *   **分类组织**: 文章分类 (`Categories.vue`) 和 标签墙 (`Tags.vue`)。
    *   **搜索**: 全文或关键字搜索 (`Search.vue`)。
*   **资源管理**:
    *   附件/文件管理 (`Attachments.vue`)，用于管理上传的图片或文件。
*   **视觉个性化** (项目的一大亮点):
    *   **动态背景**: 可配置的 3D 场景 (碎片 Shards, 星云 Nebula, 矩阵 Matrix, 银河 Galaxy)，代码见 `settings.js`。
    *   **交互特效**: 鼠标跟随特效 (花朵, 烟花, 星星)。

## 2. 推荐的开发顺序 (个人开发路线图)

如果你是个人开发者，想要从零开始开发这样一个系统，为了避免中途重构并保持开发动力，建议按照以下**五个阶段**进行：

### 第一阶段：基建与后端 (骨架)
*在写 UI 之前，你需要先有数据结构。*
1.  **需求分析与数据库设计**: 设计 `Users` (用户), `Articles` (文章), `Comments` (评论), `Tags` (标签), `Categories` (分类), 和 `Files` (文件) 等表结构。
2.  **后端 API 开发** (Java/Node/Go):
    *   优先实现 **用户认证** (登录/注册/JWT)。如果不先做这个，你就没法测试“我的主页”等功能。
    *   实现 **文章的 CRUD** (增删改查) 接口。
    *   **文件上传 API**: 这是 Markdown 编辑器上传图片所必需的。

### 第二阶段：前端基建 (框架)
3.  **项目初始化**: 使用 `npm create vite@latest` 创建项目，配置代码规范，设置路径别名 (`@/`)。
4.  **核心状态与网络层**:
    *   封装 `axios`: 配置请求拦截器（自动带 Token）和响应拦截器（处理 401 Token 过期，这正是你 `todo.md` 里提到的痛点）。
    *   设置 `UserStore` (Pinia): 用于管理用户的登录状态。
5.  **基础布局 (外壳)**:
    *   实现 `App.vue` 的整体结构：包括 导航栏 (Navbar)、侧边栏 或 抽屉，以及 `RouterView`。
    *   **路由配置**: 先定义好首页、登录页、编辑器等页面的空路由。

### 第三阶段：核心业务逻辑 (血肉)
6.  **认证 UI**: 优先完成登录/注册页面，这样你才能获取 Token 并调试接口。
7.  **文章展示 (只读模式)**:
    *   开发 **首页** (文章列表/Feed流)。
    *   开发 **文章详情页** (Markdown 渲染)。
    *   *理由*：最先看到数据渲染在屏幕上会给你带来很大的成就感。
8.  **文章创作 (编辑器)**:
    *   集成 Markdown 编辑器 (`Vditor` 等)。
    *   打通编辑器内的 **图片上传** 功能。

### 第四阶段：增强与交互 (灵魂)
9.  **社交功能**: 评论、点赞、个人资料编辑。
10. **内容组织**: 分类页、标签页和归档页（Timeline）。
11. **视觉打磨 (Wow Factor)**:
    *   **现在**才是引入 `Three.js` 背景和 `GSAP` 进场动画的最佳时机。如果在早期做这些，很容易因为调试视觉效果而忽略核心逻辑。
    *   实现全局的 `SettingsStore`，让用户可以在不同特效间切换。

### 第五阶段：优化与生产 (打磨)
12. **附件管理**: 开发 `Attachments.vue` 来管理所有上传的文件。
13. **国际化 (i18n)**: 将硬编码的字符串提取到语言包中。
14. **性能优化**: 路由懒加载、图片资源优化、打包体积分析。

---

### 总结
根据文件结构，当前项目大约处于 **第四/五阶段**。功能已经非常丰富，但正如 `todo.md` 中记录的，目前需要重点解决集成问题（如 Token 过期处理）和用户体验细节（上传限制）。
